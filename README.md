# Project Chimera

Project Chimera is an AI-native game development project for an original co-op dungeon-crawler experience built around rules, consequence, humor, spectacle, player trust, procedural systems, and adversarially audited development.

The project has two intertwined products:

1. **Project Chimera**: the game.
2. **Chimera Development System (CDS)**: the AI-native engineering and design methodology used to build the game.

## Inspiration Boundary

Project Chimera may be inspired by the emotional appeal of works like *Dungeon Crawler Carl*: absurd danger, dark humor, theatrical systems, game-like progression, and consistent rules under pressure.

Project Chimera must remain an original universe with original names, lore, characters, mechanics, factions, dialogue, quests, art direction, and implementation.

## Core Idea

Players awaken inside an impossible alien megastructure where survival is part dungeon crawl, part game show, part living simulation, and part social experiment. The world may be cruel, hilarious, theatrical, and strange, but it is not arbitrary. Rules exist. Consequences persist. Exceptions have reasons.

## Operating Principle

We are not documenting vibes. We are engineering decisions.

Every feature must answer:

1. Is it fun?
2. Does it create stories?
3. Does it make the world feel alive?
4. Does it preserve player trust?
5. Does it follow established rules, or break them for a discoverable reason?

## Completion Philosophy

No meaningful task exits its loop until every assigned greenfield auditor independently reaches **10/10** under the current scope, all findings are closed, and all verification criteria pass.

A score below 10 is not a shame event. It is a work queue.

## Current Status

Foundation phase. This repository begins as a living specification and will grow into a buildable Unreal project and AI-native development system.
